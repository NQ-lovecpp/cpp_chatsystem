## 🎯 已知限制

### 架构层面
- ⚠️ Elasticsearch 资源占用较大（适合 4c8g+ 服务器）
- ⚠️ MySQL + ES 双写存在一致性风险
- ⚠️ httplib.h 并发性能有限（~10K QPS）

### 功能层面
- ⚠️ 不支持消息撤回
- ⚠️ 不支持消息引用回复
- ⚠️ 不支持已读回执
- ⚠️ 不支持大文件分片上传
- ⚠️ 不支持多端同步

### 安全层面
- ⚠️ WebSocket 未加密（ws://）
- ⚠️ 文件上传无病毒扫描
- ⚠️ 密码使用简单哈希

## 🛠️ 部署说明

### 系统要求

**最低配置**（开发环境）：
- CPU: 2 核
- 内存: 4GB
- 磁盘: 20GB
- 系统: Ubuntu 20.04+

**推荐配置**（生产环境）：
- CPU: 4 核
- 内存: 8GB
- 磁盘: 50GB
- 系统: Ubuntu 20.04+

### 依赖服务
```
MySQL:          5.7+
Redis:          6.0+
Elasticsearch:  7.x
RabbitMQ:       3.8+
Etcd:           3.4+
Docker:         20.10+
```

### 快速部署

```bash
# 1. 克隆项目
git clone https://github.com/YOUR_USERNAME/cpp_chatsystem.git
cd cpp_chatsystem

# 2. 启动后端服务（Docker）
cd ChatSystem-Backend
docker-compose up -d

# 3. 编译 Qt 客户端
cd ../ChatSystem-Frontend-QtProj/ChatClient_Qt
mkdir build && cd build
cmake ..
make -j4

# 4. 运行客户端
./ChatClient_Qt
```

详细部署文档请参考：`ChatSystem-Backend/README.md`
